### 4. 排除使用者自己電腦的問題（不影響學生教學包本身）
- **VS Code 終端機被改成 Git Bash**：因為安裝腳本會把 `terminal.integrated.defaultProfile.windows` 設成 Git Bash（為了讓學生環境和課程一致），連使用者自己這台也被改到。已依使用者要求**還原**成 PowerShell 預設（只刪那一行，Claude Code 擴充相關設定不動）。
- **Ctrl+J 換行變成「換頁」**：診斷出是 VS Code 預設把 `Ctrl+J` 綁給 `workbench.action.togglePanel`（切換底部面板），會攔截掉按鍵、傳不到 Claude Code。已在使用者的 VS Code `settings.json` 加入：
  ```json
  "terminal.integrated.commandsToSkipShell": ["-workbench.action.togglePanel"]
  ```
  （負號寫法：取消該指令攔截，讓 Ctrl+J 改為傳給終端機。）JSON 已驗證有效，**需要使用者重開整個 VS Code 才會生效**。
- **Shift+Enter 換行沒作用**：診斷出使用者尚未執行過 `/terminal-setup`（VS Code 的 `keybindings.json` 不存在，代表這個指令沒設定過 Shift+Enter 的特殊序列）。已請使用者在 Claude Code 對話框內執行 `/terminal-setup` 指令。
- ⏳ **待確認**：使用者重開 VS Code 並跑完 `/terminal-setup` 後，Ctrl+J 與 Shift+Enter 是否都恢復正常，尚未回報結果。
